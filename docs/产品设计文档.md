# PeachAlgo Coach — 产品设计文档

> 本地优先的开源刷题教练：根据水平、目标与语言生成学习路线，再按每日做题反馈动态调整后续计划。

| 项目 | 说明 |
|------|------|
| 产品名 | PeachAlgo Coach（中文名：黄桃算法教练） |
| 版本 | v0.1（MVP 设计） |
| 定位 | 本地部署的个性化刷题规划与复盘工具 |
| 技术栈 | Vue 3 + Vite + TypeScript + FastAPI + SQLite |
| 分发方式 | GitHub 开源，本地 Web / Docker 一键启动 |
| License（建议） | MIT |

---

## 1. 背景与问题

LeetCode 题量大、标签杂、官方难度粗，常见痛点：

1. 不知道从哪开始，也排不出适合自己的顺序  
2. 静态题单无法根据真实完成情况调整  
3. 做过就忘，缺少复习机制  
4. 每天题量不合理，容易劝退  
5. 自评水平不准，导致路线过难或过简单  

本产品不替代 LeetCode 判题，而是做 **刷题教练 / 计划器**：解决“刷什么、何时刷、刷完怎么调”。

---

## 2. 产品目标

### 2.1 核心目标

让用户在本地完成：

```text
画像（水平 / 目标 / 语言 / 时间）
  → 生成学习路线
  → 领取今日任务
  → 提交做题反馈
  → 动态调整后续计划 + 安排复习
```

### 2.2 非目标（MVP 明确不做）

- 在线代码编辑器 / 判题 OJ  
- 爬取或搬运 LeetCode 题面全文  
- 强制云端账号与多用户 SaaS  
- 社交排行榜  
- 默认接入大模型（可后期做成可选插件）  

### 2.3 成功标准（MVP）

- 新用户 3 分钟内完成 onboarding 并看到今日任务  
- 完成 1 道题反馈后，次日计划可发生可解释的变化  
- 本地一条命令或 Docker 可启动  
- 数据可导出备份，默认不上传任何服务器

## 3. 用户与场景

### 3.1 目标用户

| 用户 | 诉求 |
|------|------|
| 校招准备者 | 60–90 天系统刷题，覆盖高频题型 |
| 社招转岗 / 面试者 | 短周期冲刺，偏中高频 Medium |
| 基础巩固者 | 从 Easy 模板题稳步建立知识图谱 |
| 开源自部署用户 | 数据本地、可定制题单、可二次开发 |

### 3.2 核心场景

1. **首次使用**：选语言与目标 → 可选诊断 → 生成 30/60/90 天路线  
2. **每日打开**：看今日 2–4 题、预估耗时、推荐理由  
3. **做完反馈**：难度、耗时、是否看题解 → 更新掌握度  
4. **卡住/连败**：系统自动降难度、补前置、减题量  
5. **复习到期**：插入间隔重复题，防止遗忘  
6. **周复盘**：查看标签掌握度、完成率、薄弱项  

---

## 4. 产品原则

1. **本地优先**：SQLite 单文件，隐私默认安全  
2. **宁少勿多**：每日任务可完成，避免计划羞耻  
3. **可解释推荐**：每题都说明“为什么今天推它”  
4. **反馈闭环**：没有反馈就不谈智能  
5. **版权克制**：仓库只存题号、标题、标签、链接等元数据  
6. **贡献友好**：题库 JSON/YAML 可被 PR 扩展

## 5. 信息架构与页面

### 5.1 页面结构

```text
/onboarding          首次引导（水平/目标/语言/时间）
/today               今日任务（默认首页）
/plan                总路线与阶段进度
/review              复习队列
/stats               掌握度与趋势
/history             做题历史
/problems            题库浏览（只读 + 收藏）
/settings            配置、导出导入、重置
```

### 5.2 关键页面说明

#### A. Onboarding

分步收集：

1. 主攻语言：`Java` / `Python`  
2. 当前水平：零基础 / 能做 Easy / 能独立做部分 Medium / 冲 Hard  
3. 目标：校招 / 社招 / 巩固基础 / 竞赛入门  
4. 每天可投入：30 / 60 / 90 / 120 分钟  
5. 计划周期：30 / 60 / 90 天  
6. 可选薄弱项：数组、链表、树、DP…  
7. 可选：5–8 题诊断（用于校准起点，可跳过）  

完成后生成初始计划，跳转 `/today`。

#### B. 今日任务 `/today`

展示：

- 日期、连续打卡天数  
- 今日题列表（题号、标题、难度、标签、预估时间、推荐理由）  
- 每题操作：`开始`（外链 LeetCode）、`提交反馈`、`跳过/改天`  
- 今日进度条与鼓励文案  

#### C. 反馈弹窗

必填：

- 结果：独立 AC / 看提示后 AC / 看题解后 AC / 未完成  
- 主观难度：太简单 / 刚好 / 偏难 / 很难  
- 耗时（分钟）  

选填：

- 是否想复习  
- 简短笔记  

提交后即时更新标签掌握度，并提示“已影响后续计划”。

#### D. 路线 `/plan`

- 阶段列表（基础 → 数组字符串 → … → DP）  
- 每阶段完成度  
- 当前所在节点  
- 允许“重新生成计划”（需确认）  

#### E. 统计 `/stats`

- 全局完成率、独立 AC 率  
- 各标签掌握度（0–100）  
- 近 7/30 天刷题量与均耗时  
- 薄弱 Top3 与建议

## 7. 学习路线设计

### 7.1 默认阶段顺序

```text
0. 语法与复杂度基础（按语言给模板清单）
1. 数组 / 字符串 / 哈希
2. 双指针 / 滑动窗口
3. 栈 / 队列
4. 链表
5. 二叉树 / DFS / BFS
6. 堆 / 优先队列
7. 二分查找
8. 回溯
9. 贪心
10. 图论基础
11. 动态规划（入门 → 中等）
12. 面试综合高频
```

Java / Python **共用阶段与题单**，差异仅在：

- 语言模板文档  
- 个别实现提示（如堆、有序结构 API）  

### 7.2 每阶段题目分层

| 层级 | 占比建议 | 说明 |
|------|----------|------|
| 必刷模板题 | 40% | 建立范式 |
| 巩固变体题 | 40% | 迁移能力 |
| 拔高题 | 10–20% | 可选，时间不够可跳过 |
| 复习题 | 动态插入 | 不占阶段固定配额 |

### 7.3 初始计划生成算法（规则版）

输入：`profile + problem_catalog`

步骤：

1. 按 `level/goal` 选择起点阶段与难度上限  
2. 过滤题库：去掉远超水平的 Hard（除非 advanced）  
3. 弱项标签提高权重  
4. 按阶段顺序展开题目，生成 `PlanItem` 队列  
5. 用 `daily_minutes` 做装箱：每天 2–4 题，总预估时间 ≈ 0.8–1.0 × 日预算  
6. 预留每天 0–1 个复习位（前几天为空）  
7. 写入 Plan，状态 `active`  

### 7.4 目标画像对题型权重的影响（示例）

| 目标 | 权重倾向 |
|------|----------|
| 校招 | 数组字符串、树、DP 入门、高频 Medium |
| 社招 | 中高频 Medium、系统设计向算法较少、工程常见题 |
| 巩固基础 | Easy 模板 + 少量 Medium，进度更慢 |
| 竞赛入门 | 更早引入构造/组合/较难 DP（MVP 可弱化） |

## 8. 自适应引擎（核心差异点）

### 8.1 单次表现分

每次 Attempt 计算 `performance ∈ [-2, +2]`：

```text
base:
  solved_solo      +1.5
  solved_hint      +0.5
  solved_solution  -0.5
  failed           -1.5

felt_difficulty:
  too_easy         +0.5
  ok               0
  hard             -0.5
  very_hard        -1.0

time:
  minutes <= 0.7 * est   +0.3
  minutes >= 1.5 * est   -0.3
```

截断到 `[-2, 2]`。

### 8.2 标签掌握度更新

```text
mastery_new = clamp(mastery_old + performance * 6, 0, 100)
```

同时更新全局 `rating`：

```text
rating_new = rating_old + round(performance * 12)
```

### 8.3 计划调整规则

| 信号 | 动作 |
|------|------|
| 近 3 题同标签 performance 均值 ≥ 1.0 | 跳过后续重复模板题，进入变体/下一阶段 |
| 近 3 题均值 ≤ -0.7 | 插入 1–2 道更易前置题；延后拔高题 |
| 连续 2 天未完成今日任务 | 次日题量 -1，总预估时间降到 70% |
| 连续 3 天全勤且均耗时偏低 | 次日可 +1 题或提高一档难度 |
| 某标签 mastery < 40 且 due 复习存在 | 优先插入该标签复习/巩固 |
| 用户手动跳过某题 2 次 | 降优先级或移出近 7 日窗口 |

### 8.4 可解释文案示例

- “今天做滑动窗口，因为昨天数组掌握度一般，且该阶段还差模板题。”  
- “降低题量：你连续两天超时未完成，先保证节奏。”  
- “插入复习：#3 最长子串 已到复习日。”

## 9. 复习系统（SRS）

采用简化 SM-2 思想：

### 9.1 首次进入复习池

当结果为任意 AC（含看题解）时创建 ReviewCard：

- 初始 `interval_days = 1`  
- `due_at = tomorrow`  
- `ease = 2.5`  

### 9.2 复习反馈映射

| 复习表现 | 间隔变化 |
|----------|----------|
| 很快独立想起 | interval * ease，ease +0.1 |
| 想起来但慢 | interval * 1.4 |
| 基本忘了 / 看题解 | interval = 1，ease -0.2（最低 1.3） |

默认复习点：1 / 3 / 7 / 14 / 30 天附近浮动。

### 9.3 每日混排

```text
今日任务 =
  到期复习（最多 1–2 题）
  + 主线新题（填满时间预算）
  + 弱项补强（若仍有空余且弱项明显）
```

## 11. 题库策略

### 11.1 MVP 题量

- 精选 **120–180 题** 足够跑通 30–90 天  
- 覆盖默认 12 个阶段  
- 来源可参考公开题单结构（如 Blind/NeetCode 思路），但自行整理元数据  

### 11.2 `problems.json` 示例

```json
{
  "version": 1,
  "problems": [
    {
      "id": "lc-1",
      "number": 1,
      "title": "Two Sum",
      "title_zh": "两数之和",
      "difficulty": "Easy",
      "tags": ["array", "hash-table"],
      "prerequisites": ["hash-table-basics"],
      "est_minutes": 20,
      "is_template": true,
      "stage": "array-hash",
      "url": "https://leetcode.com/problems/two-sum/",
      "priority": 10
    }
  ]
}
```

### 11.3 版权边界

- ✅ 题号、标题、难度、标签、链接、自编推荐理由  
- ❌ 完整题目描述、官方测试用例全文、付费题解搬运  

### 11.4 社区贡献

提供 `docs/contribute-problems.md`：

- 如何新增题目元数据  
- 标签枚举表  
- PR 检查清单（字段完整、阶段合理、无题面全文）

## 12. 本地部署体验（开源关键）

### 12.1 开发者启动

```bash
# backend
cd backend
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\\Scripts\\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# frontend
cd frontend
npm install
npm run dev
```

浏览器打开前端地址（如 `http://localhost:5173`）。

### 12.2 普通用户（Docker）

```bash
docker compose up --build
```

访问 `http://localhost:5173`（或文档写明的端口）。

### 12.3 数据位置

- SQLite：`backend/data/coach.db`（或 compose volume）  
- 备份：设置页一键导出 JSON  
- 重置：删除 db / 调用重置接口（需二次确认）

## 13. UX 细节

1. 今日任务默认不超过 **4 题**  
2. 每题显示预估时间，顶部显示“预计总时长”  
3. 反馈表单 ≤ 3 个必填项，手机也可快速点完  
4. 连续失败时文案鼓励，不惩罚性弹窗  
5. 外链打开 LeetCode，本产品不嵌入刷题页  
6. 支持中文界面（开源 README 中英均可，UI 先中文）  
7. 空状态都有下一步 CTA（去 onboarding / 去今日任务）

## 14. 里程碑

- 前后端打通

- 反馈提交

- 复习队列  
- 导出导入

- 诊断题校准 rating  
- Java/Python 模板页  
- 主题切换

## 15. 测试策略

| 类型 | 范围 |
|------|------|
| 单元测试 | performance 计分、mastery 更新、每日装箱、SRS 间隔 |
| 集成测试 | onboarding → today → attempt → 新 today |
| 手工验收 | Windows 本地启动、Docker 启动、导出导入 |

关键断言示例：

- 日预算 60 分钟时，今日 `sum(est_minutes)` 不超过约 60–70  
- 连续 too_easy 后，后续同标签模板题减少  
- 复习 due 当天会出现在 today
